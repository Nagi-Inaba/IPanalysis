## Slide 3: Before — Python スクリプト時代

### やっていたこと

- CKSwebから CSV をダウンロード
- Python スクリプトで CSV を整理・集計
- 特許分類増減率の計算、出願人カウントなどを pandas で処理

### 毎回手作業だったこと

- Excel でテーブルを範囲選択してグラフ化
- グラフの書式設定（軸ラベル、色、凡例）を毎回手動
- 分析のたびに同じ作業を繰り返す

### 課題

- 集計ロジックは自動化できたが、**可視化が毎回手作業**
- 分析結果を他人に共有するのが面倒（Excel ファイルを渡すしかない）
- コードが1ファイル1,000行超に肥大化

## Slide 4: After — Claude Code で Web アプリ化

### Claude Code でやったこと

1. **Streamlit で Web UI 化**

   - ファイルアップロード → 前処理 → 集計 → グラフ の3ステップ
   - ブラウザで完結、Excel 手作業ゼロ
2. **Altair でグラフ自動生成**

   - 15種以上のインタラクティブチャートを自動描画
   - ツールチップ、ズーム、フィルタが標準装備
3. **データ形式の自動判定**

   - CKSweb / J-PlatPat / Questel の列名パターンで自動検出
   - 汎用 CSV も手動列マッピングで対応
4. **出願人名寄せの内蔵**

   - JSON 辞書ベースで表記揺れを統一
   - GUI で編集・インポート・エクスポート可能
5. **1,069行 → 4ファイル分割リファクタリング**

   - app.py (455行) + charts.py (555行) + cached_agg.py (70行) + constants.py (44行)

## Slide 5: 技術スタック詳細

### コア

| 技術                  | 用途       | 選定理由                                       |
| --------------------- | ---------- | ---------------------------------------------- |
| **Python 3.8+** | 言語       | 特許分析ライブラリとの親和性                   |
